Eventiya – Event Management & Registration System
1. Introduction
1.1 Project Overview
Eventiya is a web-based Event Management and Registration System that enables event organizers to create, manage, and monitor events while allowing attendees to discover events, book tickets, complete online payments, and validate entry via QR codes. The system centralizes ticketing, payments, notifications, and reporting into a unified platform.
1.2 Objectives
Digitize event management and ticketing processes
Reduce manual errors and fraud
Provide secure online booking and payments
Enable reliable QR-based ticket validation
Deliver analytics and reporting capabilities

3. Functional Requirements (FR)
FR1 – Authentication & Authorization
Users shall be able to register and log in
System shall support role-based access control
Users shall be able to reset passwords
FR2 – User Profile Management
Users shall be able to view/edit profiles
System shall store user information securely
FR3 – Event Catalog
System shall display available events
Users shall be able to search/filter events
System shall provide event detail pages
FR4 – Event Management (Organizer)
Organizers shall be able to create events
Organizers shall be able to edit/delete events
Organizers shall define ticket pricing and limits
Organizers shall publish/unpublish events
FR5 – Ticketing & Booking
Users shall select tickets
System shall check ticket availability
System shall temporarily lock inventory
System shall confirm bookings upon payment success
FR6 – Payment Processing
System shall integrate secure payment gateway
System shall track payment status
System shall handle payment failures
FR7 – QR Code Generation
System shall generate unique QR per ticket
QR shall contain secure validation data
FR8 – Ticket Validation
System shall validate tickets via web page
Validation shall check ticket existence
Validation shall verify payment success
Validation shall prevent duplicate usage
System shall log validation attempts
FR9 – Notifications
System shall send booking confirmation emails
System shall send payment receipts
System shall send event reminders
FR10 – Reports & Analytics
Organizer
View ticket sales reports
View attendance statistics
Admin
View basic platform metrics
View basic revenue analytics
FR11 – Admin Panel
Manage users
Monitor events
Monitor payments
View logs

NFR5 – Availability

NFR6 – Usability

7. Risk Assessment
R1 – Payment Gateway Failure
Risk: Payment service downtime may block bookings Impact: High Mitigation: Implement retry logic and failure handling
R2 – Ticket Overbooking
Risk: Concurrent bookings exceed inventory Impact: High Mitigation: Use Redis-based temporary locks
R3 – QR Code Fraud / Duplication
Risk: Ticket reuse or manipulation Impact: High Mitigation: Unique tokens + usage flag validation
R4 – System Downtime
Risk: Server outage affects users Impact: Medium Mitigation: Monitoring and restart mechanisms
R5 – Data Loss
Risk: Database/storage failure Impact: High Mitigation: Regular backups
R6 – Security Breach
Risk: Unauthorized access or data theft Impact: High Mitigation: HTTPS, encryption, secure authentication
R7 – Schedule Overrun
Risk: Project delays due to scope/bugs Impact: Medium Mitigation: Agile sprint planning and prioritization
R8 – Integration Issues
Risk: Failures between services (payment/email/QR) Impact: Medium Mitigation: Integration testing
